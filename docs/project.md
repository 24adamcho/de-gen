# Component-based design.md builder script
Design.md is a format created to create and port look-and-feel style guides for website and application frontends. The specification is designed to be both machine and human friendly. However, the mix of YAML and AI prompt syntax results in a very fat format that is a pain in the butt to type out by hand, and is primarily only written by AI for AI, causing a distasteful waste of energy.
Solution: Write separate markdown files for 'components' of a design.md document, then assemble them into a full document for prompting. This not only stops the written side wasting energy, but also creates an internal ecosystem for an organization to build a 'corpus' of components, making transferrable components that don't have to be rewritten but instead browsed through. This mirrors modern architechtural firms, where they do not make some 500 building design documents from scratch, rather instead copying blueprints of every stud and bolt between projects.
Projects change, components do not. Structures change, materials do not.
